# PWA Conversion Design

## Context

MLBHQ is a React 18 SPA (Vite + Express) with zero PWA infrastructure. Users want to install it as an app on their phone/desktop and have cached data available offline. No push notifications needed.

## Approach

Use `vite-plugin-pwa` with Workbox `generateSW` mode. The plugin handles manifest generation, service worker creation, and auto-registration. Silent auto-update (no user prompt). No offline UI indicator.

## Service Worker Caching Strategy

Mode: `generateSW` (fully auto-generated, no custom SW code).

| Resource Type | Strategy | Config |
|---|---|---|
| App shell (JS, CSS, HTML) | Precache | Auto — built assets are precached at SW install |
| Images / static assets | CacheFirst | maxEntries: 60, maxAgeSeconds: 30 days |
| `/api/*` responses | StaleWhileRevalidate | maxEntries: 50, maxAgeSeconds: 3600 (1 hour) |
| External images (MLB logos from `mlbstatic.com`) | CacheFirst | maxEntries: 60, maxAgeSeconds: 7 days |

### Offline behavior by tab

| Tab | Offline behavior |
|---|---|
| Overview | Shows last-fetched stats, next game, standings |
| Roster | Shows last-fetched roster with injury data |
| Standings | Shows last-fetched standings |
| Leaders | Shows last-fetched leaderboards |
| Schedule | Shows last-fetched schedule |
| Scores (live) | Shows stale scoreboard, no live updates |

## Update Strategy

`autoUpdate` registration type — the service worker silently updates in the background when a new version is detected. No user-facing prompt or banner.

### No changes needed

- **`src/App.jsx`** — no offline banner or update prompt
- **`server/`** — no backend changes
- **`vercel.json`** — SPA rewrite already handles routing; SW files are static assets served by Vercel automatically
- **`api/`** — no serverless function changes

## Verification

1. Run `npm run build` and verify `dist/` contains `sw.js`, `manifest.webmanifest`, and icon files
2. Run `npm run preview` and open in Chrome
3. Open DevTools > Application tab:
   - Manifest section shows correct metadata and icons
   - Service Workers section shows active SW
   - Cache Storage shows precached assets and runtime caches
4. Lighthouse PWA audit scores green on installability checks
5. Test install: click browser install button, verify app launches in standalone window
6. Test offline: enable airplane mode in DevTools, reload — app shell loads, cached API data appears
7. Test update: rebuild, verify SW updates silently on next visit
